# 第 0 课：课程概览与环境搭建

## Druid 是什么？

Druid 是阿里巴巴开源的一款数据库连接池组件，但它远不止于此。Druid 内置了**强大的 SQL 解析引擎**，能够对 SQL 进行词法分析、语法分析、AST 构建、格式化、参数化、统计等操作。

> 你正在做的 "SQL to ES DSL" 本质上也是 SQL 解析的一种应用 —— 读懂 SQL，再翻译成另一种语言。这正是 Druid SQL 模块最擅长的。

## Druid SQL 处理的核心流程

一条 SQL 字符串在 Druid 中经历以下旅程：

```
原始 SQL → [Lexer 词法分析] → Token 流 → [Parser 语法分析] → AST → [Visitor 访问者] → 目标输出
```

```
"SELECT * FROM users WHERE id = 1"
         ↓
     Lexer (词法分析器)
         ↓
  [SELECT, *, FROM, users, WHERE, id, =, 1]  ← Token 序列
         ↓
     Parser (语法分析器)
         ↓
     SQLSelectStatement                  ← AST (抽象语法树)
       ├── SQLSelect
       │   ├── SQLSelectQueryBlock
       │   │   ├── selectList: [SQLAllColumnExpr]
       │   │   ├── from: SQLTableSource (users)
       │   │   └── where: SQLBinaryOpExpr (id = 1)
       │   └── ...
         ↓
     Visitor (访问者)
         ↓
  格式化SQL / 统计信息 / ES DSL / 方言转换...
```

## Druid SQL 模块架构总览

```
druid/core/src/main/java/com/alibaba/druid/sql/
│
├── SQLUtils.java          ★ 统一入口，提供 format/parse/toSQLString 等方法
├── SQLDialect.java         方言定义
│
├── parser/                 ★ 解析引擎
│   ├── Lexer.java          词法分析器
│   ├── Token.java          Token 枚举 (所有 SQL 关键字)
│   ├── SQLParser.java      解析器基类
│   ├── SQLExprParser.java  表达式解析器
│   ├── SQLStatementParser.java  语句解析器
│   └── SQLSelectParser.java    SELECT 解析器
│
├── ast/                    ★ 抽象语法树节点
│   ├── SQLObject.java      AST 节点根接口
│   ├── SQLExpr.java        表达式接口
│   ├── SQLStatement.java   语句接口
│   ├── expr/               72 种表达式节点
│   └── statement/          284 种语句节点
│
├── visitor/                ★ 访问者模式实现
│   ├── SQLASTVisitor.java  访问者接口
│   ├── SQLASTOutputVisitor.java  输出 SQL 字符串
│   └── SchemaStatVisitor.java    收集表/列/条件统计
│
├── dialect/                各数据库方言 (30种)
│   └── mysql/parser/       MySQL 解析器 (最佳参考)
│       ├── MySqlLexer.java
│       ├── MySqlExprParser.java
│       ├── MySqlStatementParser.java
│       └── MySqlSelectParser.java
│
├── builder/                SQL 编程式构建
└── repository/             Schema 元数据管理
```

### Hello Druid SQL

创建一个最简单的示例，体验 SQL 解析的全过程：

```java
import com.alibaba.druid.DbType;
import com.alibaba.druid.sql.SQLUtils;
import com.alibaba.druid.sql.ast.SQLStatement;
import com.alibaba.druid.sql.ast.statement.SQLSelectStatement;
import java.util.List;

public class DruidSqlHelloWorld {
    public static void main(String[] args) {
        String sql = "SELECT id, name FROM users WHERE age > 18 ORDER BY id DESC";

        // 1. SQL 格式化
        String formatted = SQLUtils.format(sql, DbType.mysql);
        System.out.println("=== 格式化后的 SQL ===");
        System.out.println(formatted);

        // 2. 解析为 AST
        List<SQLStatement> stmts = SQLUtils.parseStatements(sql, DbType.mysql);
        SQLStatement stmt = stmts.get(0);
        System.out.println("\n=== AST 类型 ===");
        System.out.println(stmt.getClass().getSimpleName());

        // 3. 再转回 SQL 字符串
        String toString = SQLUtils.toSQLString(stmt, DbType.mysql);
        System.out.println("\n=== AST → SQL ===");
        System.out.println(toString);

        // 4. 添加条件
        String withCondition = SQLUtils.addCondition(sql, "status = 1", DbType.mysql);
        System.out.println("\n=== 添加条件后 ===");
        System.out.println(withCondition);
    }
}
```
