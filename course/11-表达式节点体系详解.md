# 第 11 课：表达式节点体系详解

## 表达式分类

Druid 的 72 种表达式节点可以分为以下几大类：

```
表达式 (SQLExpr)
├── 字面量 (Literals)
│   ├── SQLIntegerExpr     -- 1, 100, -5
│   ├── SQLCharExpr        -- 'hello', 'world'
│   ├── SQLBooleanExpr     -- true, false
│   ├── SQLNullExpr        -- NULL
│   ├── SQLFloatExpr       -- 3.14
│   ├── SQLDoubleExpr      -- 1.23e10
│   ├── SQLDecimalExpr     -- 123.456 (高精度)
│   └── SQLHexExpr         -- 0x1A2B
│
├── 标识符 (Identifiers)
│   ├── SQLIdentifierExpr  -- id, name, table_name
│   ├── SQLPropertyExpr    -- t.id, schema.table.col
│   └── SQLAllColumnExpr   -- *
│
├── 运算 (Operations)
│   ├── SQLBinaryOpExpr    -- a + b, age > 18, id = ?
│   ├── SQLUnaryExpr       -- -x, NOT flag, +val
│   └── SQLBinaryOpExprGroup -- 连续 AND/OR 分组(性能优化)
│
├── 函数调用 (Functions)
│   ├── SQLMethodInvokeExpr  -- NOW(), SUBSTR(x,1,3)
│   ├── SQLAggregateExpr     -- COUNT(*), SUM(x), AVG(y)
│   └── SQLCastExpr          -- CAST(x AS INT)
│
├── 条件/逻辑 (Conditions)
│   ├── SQLBetweenExpr     -- x BETWEEN a AND b
│   ├── SQLInListExpr      -- x IN (1,2,3)
│   ├── SQLInSubQueryExpr  -- x IN (SELECT ...)
│   ├── SQLExistsExpr      -- EXISTS (SELECT ...)
│   ├── SQLCaseExpr        -- CASE WHEN ... THEN ... END
│   └── SQLLikeExpr        -- name LIKE '%abc%'
│
└── 特殊 (Special)
    ├── SQLVariantRefExpr  -- @var (变量引用)
    ├── SQLListExpr        -- (1, 2, 3) (列表)
    ├── SQLQueryExpr       -- (SELECT ...) (子查询表达式)
    └── SQLDefaultExpr     -- DEFAULT (默认值)
```

## 1. 字面量

```java
// SQLIntegerExpr: id = 42
SQLIntegerExpr intExpr = new SQLIntegerExpr(42);
// 获取值: intExpr.getNumber() → 42
// 计算类型: intExpr.computeDataType() → BIGINT

// SQLCharExpr: name = 'Alice'
SQLCharExpr strExpr = new SQLCharExpr("Alice");
// 获取值: strExpr.getText() → "Alice"

// SQLBooleanExpr: active = true
SQLBooleanExpr boolExpr = new SQLBooleanExpr(true);
// 获取值: boolExpr.getValue() → true
```

## 2. 标识符

```java
// SQLIdentifierExpr: 简单列名或表名
SQLIdentifierExpr colRef = new SQLIdentifierExpr("id");
colRef.getName();  // "id"

// ★ 名称可能被 normalize（去除引号）
colRef.getSimpleName();  // "id" (已标准化)

// SQLPropertyExpr: 限定名 t.id
SQLPropertyExpr propRef = new SQLPropertyExpr("t", "id");
propRef.getOwner().toString();    // "t"
propRef.getName();                // "id"

// SQLAllColumnExpr: *
SQLAllColumnExpr all = new SQLAllColumnExpr();
```

### 💡 normalize 机制

Druid 在处理标识符时，会去除引号（反引号、双引号等）：

```java
// SQLUtils.normalize()
SQLUtils.normalize("`users`");    // "users"
SQLUtils.normalize("\"users\"");  // "users"
SQLUtils.normalize("[users]");    // "users"
```

这确保你在处理标识符时不需要关心原始 SQL 用的是哪种引号。

## 3. 二元操作符 — 最重要的表达式节点

`SQLBinaryOpExpr` 是 Druid 中**最常见**的表达式节点。WHERE 条件、JOIN 条件、SET 赋值中的 "=" 都是用这个节点表示的。

```java
// age > 18
SQLBinaryOpExpr expr = new SQLBinaryOpExpr(
    new SQLIdentifierExpr("age"),   // 左操作数
    SQLBinaryOperator.GreaterThan,   // 操作符
    new SQLIntegerExpr(18)           // 右操作数
);
```

### SQLBinaryOperator 枚举

```java
public enum SQLBinaryOperator {
    // 算术
    Add, Subtract, Multiply, Divide, Modulus,

    // 比较
    Equality,              // =
    GreaterThan,           // >
    GreaterThanOrEqual,    // >=
    LessThan,              // <
    LessThanOrEqual,       // <=
    NotEqual,              // != 或 <>

    // 逻辑
    BooleanAnd,            // AND
    BooleanOr,             // OR
    BooleanXor,            // XOR

    // 其他
    Concat,                // ||
    Mod,                   // %
    // ...
}
```

### 嵌套的二元操作

```java
// age > 18 AND status = 'active'
// 实际上是 SQLBinaryOpExpr(AND)
//   ├── left: SQLBinaryOpExpr(>)
//   │   ├── left: SQLIdentifierExpr(age)
//   │   └── right: SQLIntegerExpr(18)
//   └── right: SQLBinaryOpExpr(=)
//       ├── left: SQLIdentifierExpr(status)
//       └── right: SQLCharExpr('active')

SQLBinaryOpExpr andExpr = (SQLBinaryOpExpr) whereExpr;
SQLBinaryOpExpr ageCond = (SQLBinaryOpExpr) andExpr.getLeft();
SQLBinaryOpExpr statusCond = (SQLBinaryOpExpr) andExpr.getRight();

System.out.println(ageCond.getOperator());    // GreaterThan
System.out.println(ageCond.getLeft());        // age
System.out.println(ageCond.getRight());       // 18
```

## 4. 函数调用

### 普通函数: SQLMethodInvokeExpr

```java
// SUBSTR(name, 1, 3)
SQLMethodInvokeExpr func = new SQLMethodInvokeExpr("SUBSTR");
func.addParameter(new SQLIdentifierExpr("name"));
func.addParameter(new SQLIntegerExpr(1));
func.addParameter(new SQLIntegerExpr(3));

func.getMethodName();  // "SUBSTR"
func.getArguments();   // [name, 1, 3]
```

### 聚合函数: SQLAggregateExpr

```java
// COUNT(DISTINCT id)
SQLAggregateExpr agg = new SQLAggregateExpr("COUNT");
agg.setOption(SQLAggregateOption.DISTINCT);
agg.addArgument(new SQLIdentifierExpr("id"));

agg.getMethodName();  // "COUNT"
agg.getArguments();   // [id]
agg.getOption();      // DISTINCT
```

## 5. CASE WHEN

```java
// CASE WHEN score >= 90 THEN 'A' WHEN score >= 80 THEN 'B' ELSE 'C' END
SQLCaseExpr caseExpr = new SQLCaseExpr();

// WHEN score >= 90 THEN 'A'
SQLCaseExpr.Item item1 = new SQLCaseExpr.Item();
item1.setConditionExpr(new SQLBinaryOpExpr(score, >=, 90));
item1.setValueExpr(new SQLCharExpr("A"));
caseExpr.addItem(item1);

// ELSE 'C'
caseExpr.setElseExpr(new SQLCharExpr("C"));
```

## 表达式解析的完整示例

```java
String sql = "SELECT id, name, age, CASE WHEN age >= 18 THEN 'adult' ELSE 'child' END AS type FROM users";
List<SQLStatement> stmts = SQLUtils.parseStatements(sql, DbType.mysql);
SQLSelectStatement selectStmt = (SQLSelectStatement) stmts.get(0);
SQLSelectQueryBlock queryBlock = selectStmt.getSelect().getQueryBlock();

// 遍历 SELECT 列表
for (SQLSelectItem item : queryBlock.getSelectList()) {
    SQLExpr expr = item.getExpr();
    System.out.println("表达式类型: " + expr.getClass().getSimpleName());

    if (expr instanceof SQLCaseExpr) {
        SQLCaseExpr caseExpr = (SQLCaseExpr) expr;
        for (SQLCaseExpr.Item caseItem : caseExpr.getItems()) {
            System.out.println("  WHEN: " + caseItem.getConditionExpr());
            System.out.println("  THEN: " + caseItem.getValueExpr());
        }
        if (caseExpr.getElseExpr() != null) {
            System.out.println("  ELSE: " + caseExpr.getElseExpr());
        }
    }
}
```

输出：
```
表达式类型: SQLIdentifierExpr
表达式类型: SQLIdentifierExpr
表达式类型: SQLIdentifierExpr
表达式类型: SQLCaseExpr
  WHEN: age >= 18
  THEN: 'adult'
  ELSE: 'child'
```

## 💡 给你的 SQL-to-ES DSL 提示

当你遍历 WHERE 条件时，常见的 SQL 到 ES 的映射关系：

```java
// SQLBinaryOpExpr(age > 18)
// → ES: { "range": { "age": { "gt": 18 } } }

// SQLBinaryOpExpr(status = 'active')
// → ES: { "term": { "status": "active" } }

// SQLBinaryOpExpr(BooleanAnd)
//   ├── left: SQLInListExpr(city IN (...))
//   └── right: SQLBinaryOpExpr(age > 18)
// → ES: { "bool": { "must": [ ... ] } }

// SQLLikeExpr(name LIKE '%abc%')
// → ES: { "wildcard": { "name": "*abc*" } }

// SQLOrderBy (ORDER BY id DESC)
// → ES: { "sort": [{ "id": "desc" }] }

// SQLLimit (LIMIT 10)
// → ES: { "size": 10 }
```

## 思考题答案

<details>
<summary>点击展开</summary>

1. **`SQLBinaryOpExpr` 中操作符为什么用枚举不用字符串？**
   - 和 Token 枚举类似的理由：类型安全（编译器检查）、性能（枚举比较）、switch 支持。用字符串的话，`"GreaterThan"` 和 `"greaterThan"` 都能通过编译，但语义完全不同。

2. **`SQLMethodInvokeExpr` 和 `SQLAggregateExpr` 在 AST 中怎么区分？**
   - 通过 `instanceof` 判断：`SQLAggregateExpr` 有 `getOption()` 方法（DISTINCT/ALL）和 `getArguments()`（参数列表，不含 * 的特殊处理），`SQLMethodInvokeExpr` 有 `getOwner()`（可能为 null）和 `getArguments()`。
   - 在解析时，`parseIdentifier()` 中通过 `isAggregateFunction()` 判断函数名是否在聚合函数列表中，如果是则创建 `SQLAggregateExpr`，否则创建 `SQLMethodInvokeExpr`。

3. **写一个方法判断是否是"等值比较"：**
   ```java
   public static boolean isEqualityCondition(SQLExpr expr) {
       if (!(expr instanceof SQLBinaryOpExpr)) return false;
       SQLBinaryOpExpr binOp = (SQLBinaryOpExpr) expr;
       if (binOp.getOperator() != SQLBinaryOperator.Equality) return false;
       SQLExpr left = binOp.getLeft();
       SQLExpr right = binOp.getRight();
       // 确保一边是列引用，一边是字面量
       boolean leftIsCol = left instanceof SQLIdentifierExpr || left instanceof SQLPropertyExpr;
       boolean rightIsLiteral = right instanceof SQLIntegerExpr
               || right instanceof SQLCharExpr || right instanceof SQLBooleanExpr;
       return leftIsCol && rightIsLiteral;
   }
   ```
